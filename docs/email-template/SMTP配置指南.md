## 🚀 详细配置步骤（以 SendGrid 为例）

### 步骤 1：注册 SendGrid 账号

1. 访问 [SendGrid 官网](https://sendgrid.com/)
2. 点击 **Start for Free** 注册账号
3. 验证邮箱地址

### 步骤 2：创建 API Key

1. 登录后进入 **Settings** → **API Keys**
2. 点击 **Create API Key**
3. 命名为 `Supabase SMTP`，权限选择 **Full Access**
4. **复制 API Key**（只显示一次，请妥善保存）

### 步骤 3：配置发件人身份

1. 进入 **Settings** → **Sender Authentication**
2. 选择 **Domain Authentication**（推荐）或 **Single Sender Verification**

**Domain Authentication（自定义域名）：**
- 需要添加 DNS 记录到您的域名
- 提供更好的信誉和送达率
- 发件人地址：`noreply@yourdomain.com`

**Single Sender Verification（单一发件人）：**
- 快速验证，适合测试
- 只需验证单个邮箱地址
- 发件人地址：`your@email.com`

### 步骤 4：在 Supabase 中配置 SMTP

1. 登录 [Supabase Dashboard](https://app.supabase.com)
2. 选择您的项目
3. 进入 **Project Settings** → **Authentication**
4. 滚动到 **SMTP Settings** 部分
5. 启用 **Enable Custom SMTP**

填入以下信息：

```plaintext
SMTP Host: smtp.sendgrid.net
SMTP Port: 587 或 465
SMTP Username: apikey （固定值，不要改）
SMTP Password: 粘贴您的 SendGrid API Key
Sender Email: noreply@yourdomain.com
Sender Name: 终末地抽卡分析器
```

**重要提示：**
- `SMTP Username` 必须填写 `apikey`（字面值）
- `SMTP Password` 填写您的 SendGrid API Key
- 端口选择：
  - `587`：TLS 加密（推荐）
  - `465`：SSL 加密
  - `25`：不加密（不推荐）

### 步骤 5：测试邮件发送

1. 保存 SMTP 配置
2. 在您的应用中注册一个测试账号
3. 检查是否收到验证邮件
4. 查看 SendGrid Dashboard 的 **Activity** 页面确认发送状态

## 🛡️ 提高邮件送达率的最佳实践

### 1. 配置 SPF 记录

在您的域名 DNS 中添加 SPF 记录：

```dns
TXT @ v=spf1 include:sendgrid.net ~all
```

### 2. 配置 DKIM 记录

SendGrid 和 Resend 会提供 DKIM 记录，添加到 DNS：

```dns
CNAME s1._domainkey s1.domainkey.u12345.wl.sendgrid.net
CNAME s2._domainkey s2.domainkey.u12345.wl.sendgrid.net
```

### 3. 配置 DMARC 记录

```dns
TXT _dmarc v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com
```

### 4. 使用自定义域名

- ✅ 使用 `noreply@yourdomain.com`
- ❌ 避免使用 `noreply@gmail.com`

### 5. 避免垃圾邮件陷阱

- 不要使用过多的大写字母
- 避免使用 "免费"、"中奖" 等敏感词
- 保持邮件模板简洁专业
- 提供清晰的退订链接

## 🔍 故障排查

### 问题 1：邮件发送失败

**检查清单：**
- ✅ SMTP 凭据是否正确
- ✅ 端口是否被防火墙阻止
- ✅ 发件人地址是否已验证
- ✅ SMTP 服务商账户是否正常

**查看日志：**
- Supabase Dashboard → Logs
- SMTP 服务商的 Activity/Dashboard

### 问题 2：邮件进入垃圾箱

**可能原因：**
- 缺少 SPF/DKIM/DMARC 记录
- 发件人域名信誉低
- 邮件内容包含敏感词
- 邮件模板格式不规范

**解决方案：**
- 配置完整的 DNS 记录
- 使用专业的邮件模板
- 避免使用短链接
- 添加退订功能

### 问题 3：发送速度慢

**可能原因：**
- 免费版限流
- 网络延迟（国内访问国外服务）
- 批量发送时触发限制

**解决方案：**
- 升级到付费版
- 使用国内 SMTP 服务（阿里云、腾讯云）
- 实施发送队列和重试机制

**A:** 大多数 SMTP 服务商提供按需付费：
- SendGrid: $19.95/月（40,000 封）

**A:** ⭐ **Resend**（强烈推荐）

3. 添加发件人到白名单提示
