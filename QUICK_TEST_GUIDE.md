## 🔍 调试技巧

### 查看队列状态（浏览器控制台）

```javascript
// 打开控制台 (F12)，输入以下代码：

// 1. 导入队列模块
import { getGlobalQueue } from './src/utils/requestQueue.js';

// 2. 获取队列实例
const queue = getGlobalQueue();

// 3. 查看队列状态
console.log('队列状态:', queue.getStatus());

// 4. 监听队列事件
queue.addListener((event, data) => {
  console.log(`[Queue Event] ${event}:`, data);
});
```

### 查看网络请求

```javascript
// 在控制台中查看所有网络请求
performance.getEntriesByType('resource').forEach(r => {
  console.log(r.name, r.duration + 'ms');
});
```

### 模拟不同网络状况

**Chrome DevTools**:
1. F12 打开开发者工具
2. Network 标签
3. 选择网络状况：
   - Online（正常）
   - Slow 3G（慢速）
   - Offline（离线）
   - Custom（自定义）

## 🐛 常见问题排查

### 问题 1: 队列状态不显示

**可能原因**:
- 导入状态不是 `AUTHENTICATING` 或 `FETCHING`
- 队列模块未正确导入

**解决方法**:
```javascript
// 检查导入状态
console.log('当前状态:', status);

// 检查队列模块
import { getGlobalQueue } from './src/utils/requestQueue.js';
console.log('队列实例:', getGlobalQueue());
```

### 问题 2: 重试信息不显示

**可能原因**:
- 网络错误未触发重试条件
- 事件监听器未正确添加

**解决方法**:
```javascript
// 检查事件监听器
const queue = getGlobalQueue();
queue.addListener((event, data) => {
  console.log('事件:', event, data);
});
```

### 问题 3: 模拟器切换卡池仍有问题

**可能原因**:
- 浏览器缓存未清除
- 代码未正确更新

**解决方法**:
```bash
# 清除缓存并重新构建
npm run build
# 或强制刷新浏览器 (Ctrl + Shift + R)
```
